Vexa installed.

This chart is designed around the upstream Vexa Docker Compose topology.

Bot orchestration in Kubernetes:
- Default in this chart: ORCHESTRATOR=process
  Bots are spawned as child Node.js processes inside the bot-manager Pod (no Docker socket).
  NOTE: this requires your bot-manager image to include the vexa-bot Node.js code/runtime + Playwright deps.

- Optional PoC: ORCHESTRATOR=kubernetes
  Bots are spawned as Kubernetes Pods via the Kubernetes API.
  Enable RBAC for bot-manager with:
    botManager.kubernetesOrchestrator.createRbac=true
